# Implementation Plan - TeruTeruShell (No PTY)

## 1. 개요
기존의 가짜 PTY(Redirect I/O) 기반 아키텍처는 교착 상태 및 외부 프로그램(agy) 출력 버퍼링 문제를 극복하지 못하여 전면 폐기되었습니다. 이에 따라 C#으로 직접 구현하는 전면 독자 쉘인 **TeruTeruShell**을 백엔드 엔진으로 채택하여 새로운 구현 계획을 수립합니다.

## 2. 기존 계획 폐기
이전 `Implementation_Plan_Revised.md`에 명시되었던 PTY 안정성 점검 및 자원 회수 관련 마일스톤은 더 이상 유효하지 않으므로 폐기합니다.

## 3. TeruTeruShell 구현 마일스톤

### Milestone 1: TeruTeruShell 코어 엔진 및 옵저버 패턴 구조 설계
- **목표**: 외부 터미널 의존성을 제거하고 직접 명령을 파싱, 실행하며 출력을 통제하는 코어 엔진 구축.
- **범위**:
  - 명령어 파서(Parser) 및 Command 인터페이스 설계.
  - 옵저버 패턴(Observer Pattern)을 적용하여 명령 실행 중 발생하는 출력을 실시간 이벤트로 방출하는 구조 구현.
  - 출력 데이터를 구조화된 JSON 객체(type, text, color, timestamp 등 포함)로 포맷팅.

### Milestone 2: 샌드박싱 및 권한 제어
- **목표**: 사용자 계정 등급에 따른 파일 시스템 접근 권한 분리 및 디렉토리 격리.
- **범위**:
  - Admin: OS 전체 파일 시스템(C:\, D:\ 등) 자유로운 접근 허용.
  - User: 서버 실행 파일 경로 하위에 `home/{username}/` 디렉토리를 자동 생성하고 해당 디렉토리 내부로 접근 제한(Chroot 격리). 상위 디렉토리(`..`) 접근 시도 차단.
  - 가상 경로 맵핑 및 경로 유효성 검증 로직 구현.

### Milestone 3: 내장 명령어 구현
- **목표**: 포괄적인 쉘 기능을 내장하여 외부 프로그램 의존 최소화.
- **범위**:
  - 파일 시스템 이동 및 조작 내장 명령어: `cd`, `dir`, `ls`, `mkdir`, `rm`, `cp`.
  - 기초 유틸리티 및 특수 명령: `ping`, 안티그래비티 전용 도구 `agy` 호출 기능.

### Milestone 4: 프론트엔드 연동
- **목표**: SignalR과 xterm.js를 이용한 프론트엔드 통합 및 실시간 스트리밍 시각화.
- **범위**:
  - 백엔드에서 전송되는 구조화된 JSON 패킷 파싱.
  - `type`이나 `color` 속성에 맞춰 ANSI Escape Code로 변환.
  - 변환된 데이터를 xterm.js 화면에 렌더링.
  - 백엔드는 순수 데이터만 다루고 화면 꾸미기는 프론트엔드가 전담하는 완벽한 역할 분리.

## 4. 진행 가이드라인
각 마일스톤은 순차적으로 진행되며, 각 단계 완료 시 충분한 단위 테스트와 통합 테스트를 거쳐 다음 단계로 넘어갑니다. 특히 샌드박싱(Milestone 2)은 보안과 직결되므로 철저한 검증이 필요합니다.
